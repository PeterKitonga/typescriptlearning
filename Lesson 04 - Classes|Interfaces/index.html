Classes & Interfaces

Classes & Interfaces

In this section I learned about classes and interfaces. These build on Object Oriented Programming concepts by working with entities that hold logic of our code. Theoretically speaking classes are blueprints for objects and help use define the structure of how our objects should be.

Defining a Class

In TypeScript, we can define a class by using the class keyword and passing the name of the class starting with a capital letter by convension. In the class we can add a constructor which initializes properties of our class. The constructor accepts arguments that the class will receive values from. We use the new keyword when calling the class so as to pass values to its constructor. 

  1. class Department {
  2.   name: string;
  3.  
  4.   constructor(n: string) {
  5.     this.name = n;
  6.   }
  7. }
  8.  
  9. const accounting = new Department('Accounting');
The 'this' keyword

In TypeScript, 'this' is a special keyword that if passed as an argument to a class method will not really require us to pass an argument when calling the method. We pass a type refering to the class so as to always ensure that this will always refer to the class. 

  1. class Department {
  2.   name: string;
  3.  
  4.   constructor(n: string) {
  5.     this.name = n;
  6.   }
  7.  
  8.   // makes sure that 'this' will always refer to the class
  9.   describe(this: Department) {
  10.     console.log(`Department: ${this.name}`);
  11.   }
  12. }
  13.  
  14. const accounting = new Department('Accounting');
  15.  
  16. accounting.describe();
The 'private' and 'public' access modifiers

These modifiers define how properties of a class are accessed outside the class. Just as the name suggests private access modifiers define properties that are only accessed within a class and cannot be modified by anything external. This helps define what can and cannot be manipulated outside the class. By default all properties are public and we do not really need to pass the public keyword. 

  1. class Department {
  2.   public name: string; // by default all properties are public
  3.   private employees: string[] = []; // will not allow access from outside the class
  4.  
  5.   constructor(n: string) {
  6.     this.name = n;
  7.   }
  8.  
  9.   // makes sure that 'this' will always refer to the class
  10.   describe(this: Department) {
  11.     console.log(`Department: ${this.name}`);
  12.   }
  13.  
  14.   addEmployee(employee: string) {
  15.     this.employees.push(employee);
  16.   }
  17.  
  18.   getEmployees() {
  19.     console.log(`Number of Employees: ${this.employees.length}`);
  20.     console.log(`Employees: ${JSON.stringify(this.employees)}`);
  21.   }
  22. }
  23.  
  24. const accounting = new Department('Accounting');
  25.  
  26. accounting.describe();
  27.  
  28. accounting.addEmployee('Jake');
  29. accounting.addEmployee('George');
  30.  
  31. accounting.getEmployees();

We can as well pass the access modifiers in our constructor as a shorthand which we don't have to initialize the local variables for the class twice. 

  1. class Department {
  2.   private employees: string[] = []; // will not allow access from outside the class
  3.  
  4.   constructor(private _id: string, public name: string) {}
  5.  
  6.   // makes sure that 'this' will always refer to the class
  7.   describe(this: Department) {
  8.     console.log(`Department #(${this._id}): ${this.name}`);
  9.   }
  10.  
  11.   addEmployee(employee: string) {
  12.     this.employees.push(employee);
  13.   }
  14.  
  15.   getEmployees() {
  16.     console.log(`Number of Employees: ${this.employees.length}`);
  17.     console.log(`Employees: ${JSON.stringify(this.employees)}`);
  18.   }
  19. }
  20.  
  21. const accounting = new Department('XiethahSh2wa6pa3hoh6ahN3', 'Accounting');
The 'readonly' modifier

The readonly modifier is used to prevent assignments to a property outside the constructor. This means you can only change its value within the constructor. 

  1. class Department {
  2.   private employees: string[] = []; // will not allow access from outside the class
  3.  
  4.   // _id is read only whose value cannot be changed outside the constructor
  5.   constructor(private readonly _id: string, public name: string) {}
  6.  
  7.   // other class logic...
  8. }
Inheritance using 'extends'

Inheritance in classes can be defined using the extends keyword where we pass the name of the class that the class in question inherits from. This means that the sub class inherits all properties of the base class including its methods and can define its own methods and properties. We can also override the properties and methods of the base class. 

  1. class Department {
  2.   private employees: string[] = []; // will not allow access from outside the class
  3.  
  4.   constructor(private readonly _id: string, public name: string) {}
  5.  
  6.   // makes sure that 'this' will always refer to the class
  7.   describe(this: Department) {
  8.     console.log(`Department #(${this._id}): ${this.name}`);
  9.   }
  10.  
  11.   addEmployee(employee: string) {
  12.     this.employees.push(employee);
  13.   }
  14.  
  15.   getEmployees() {
  16.     console.log(`Number of Employees: ${this.employees.length}`);
  17.     console.log(`Employees: ${JSON.stringify(this.employees)}`);
  18.   }
  19. }
  20.  
  21. class Engineering extends Department {
  22.   /**
  23.    * For us to initialize values through a constructor
  24.    * we must pass the super() method with arguments
  25.    * of the base class this class inherits from
  26.    */
  27.   constructor(_id: string, public designers: string[]) {
  28.     super(_id, 'Engineer');
  29.   }
  30. }
  31.  
  32. class Accounting extends Department {
  33.   constructor(_id: string, public auditors: string[], private reports: string[] = ['Audit 2020']) {
  34.     super(_id, 'Accounting');
  35.   }
  36.  
  37.   /**
  38.    * We can also add methods specific to a sub class
  39.    */
  40.   addReports(report_name: string) {
  41.     this.reports.push(report_name);
  42.   }
  43.  
  44.   getReports() {
  45.     console.log(`Number of Reports: ${this.reports.length}`);
  46.     console.log(`Report List: ${JSON.stringify(this.reports)}`);
  47.   }
  48. }
The 'protected' modifier

The protected modifier defines properties that are only visible to the class they are declared in and its sub classes. This ensures that these properties are not manipulated outside the confines of inheritance. 

  1. class Department {
  2.   protected employees: string[] = []; // will allow access within class and sub classes only
  3.  
  4.   constructor(private readonly _id: string, public name: string) {}
  5.  
  6.   // other class logic...
  7.  
  8.   addEmployee(employee: string) {
  9.     this.employees.push(employee);
  10.   }
  11.  
  12.   getEmployees() {
  13.     console.log(`Number of Employees: ${this.employees.length}`);
  14.     console.log(`Employees: ${JSON.stringify(this.employees)}`);
  15.   }
  16. }
  17.  
  18. class Accounting extends Department {
  19.   constructor(_id: string, public auditors: string[], private reports: string[] = ['Audit 2020']) {
  20.     super(_id, 'Accounting');
  21.   }
  22.  
  23.   // other class logic...
  24.  
  25.   addEmployee(name: string) {
  26.     if (name === 'Peter') {
  27.       return;
  28.     }
  29.  
  30.     // can access protected property in the base class
  31.     this.employees.push(name);
  32.   }
  33. }
Getters and Setters

Getters and setters are accessors that allow us to retrieve and update the value of a property in the class. The type of the setter parameter must match the return type of the getter. For more info check here → 

  1. class Accounting extends Department {
  2.   private last_report: string;
  3.  
  4.   constructor(_id: string, public auditors: string[], private reports: string[] = ['Audit 2020']) {
  5.     super(_id, 'Accounting');
  6.     this.last_report = reports[0];
  7.   }
  8.  
  9.   // can be used to access private properties of the class
  10.   // getters must return a value
  11.   get mostRecentReport() {
  12.     if (!this.last_report) {
  13.       throw new Error('No report found');
  14.     }
  15.  
  16.     return this.last_report;
  17.   }
  18.  
  19.   // sets value of a property in the class
  20.   set mostRecentReport(value: string) {
  21.     if (!value) {
  22.       throw new Error('Please pass a value to update last report');
  23.     }
  24.  
  25.     this.addReports(value);
  26.   }
  27.  
  28.   /**
  29.    * We can also add methods specific to a sub class
  30.    */
  31.   addReports(report_name: string) {
  32.     this.reports.push(report_name);
  33.     this.last_report = report_name;
  34.   }
  35.  
  36.   getReports() {
  37.     console.log(`Number of Reports: ${this.reports.length}`);
  38.     console.log(`Report List: ${JSON.stringify(this.reports)}`);
  39.   }
  40.  
  41.   addEmployee(name: string) {
  42.     if (name === 'Peter') {
  43.       return;
  44.     }
  45.  
  46.     // can access protected property in the base class
  47.     this.employees.push(name);
  48.   }
  49. }
  50.  
  51. const accounting = new Accounting('dahquaitah2ohph0eevooc8k', ['Jane', 'Mary']);
  52.  
  53. // getters are accessed like properties
  54. console.log(`Last Report One: ${accounting.mostRecentReport}`);
  55.  
  56. // this will call the setter
  57. accounting.mostRecentReport = 'Another Report';
  58.  
  59. console.log(`Last Report Two: ${accounting.mostRecentReport}`);
Static methods and properties

Static properties and methods are those that can be accessed without instantiating a class via the new keyword. Their values can be accessed and modified without initializing in the constructor. The can be inherited and can also be defined with the visibility modifiers public, protected, private. 

  1. class Department {
  2.   protected employees: string[] = []; // will allow access to class and subclasses only
  3.   static fiscal_year: string = '2020';
  4.  
  5.   constructor(private readonly _id: string, public name: string) {}
  6.  
  7.   // other class logic...
  8.  
  9.   getFiscalYear() {
  10.     /**
  11.      * static properties cannot be access by 'this' in a non static method, 
  12.      * we call the class to access it
  13.      */
  14.     return {
  15.       employment_year: Department.fiscal_year
  16.     };
  17.   }
  18.  
  19.   static createEmployee(name: string) {
  20.     // can be access by 'this' in a static method
  21.     return {
  22.       name,
  23.       employment_year: Department.fiscal_year
  24.     };
  25.   }
  26. }
  27.  
  28. const employee_one = Department.createEmployee('Joseph');
  29. console.log(employee_one);
  30. const year = Department.fiscal_year;
  31. console.log(year);
Abstract classes

Abstract classes are those that cannot be instantiated via the new keyword. To do that we must implement a sub class that inherits from the abstract class. For such classes we can also define abstract methods that we want all sub classes to implement. This forces sub classes to always override the abstract methods of their parent base class. 

  1. abstract class Department {
  2.   protected employees: string[] = []; // will allow access to class and subclasses only
  3.   static fiscal_year: string = '2020';
  4.  
  5.   constructor(protected readonly _id: string, public name: string) {}
  6.  
  7.   // makes sure that 'this' will always refer to the class
  8.   // abstract method is defined with a return type and no body
  9.   abstract describe(this: Department): void;
  10.  
  11.   // other class methods...
  12. }
  13.  
  14. class Engineering extends Department {
  15.   constructor(_id: string, public designers: string[]) {
  16.     super(_id, 'Engineer');
  17.   }
  18.  
  19.   describe() {
  20.     console.log(`Department Engineering #(${this._id}): ${this.name}`);
  21.   }
  22.  
  23.   // other class methods...
  24. }
  25.  
  26. class Accounting extends Department {
  27.   private last_report: string;
  28.  
  29.   constructor(_id: string, public auditors: string[], private reports: string[] = ['Audit 2020']) {
  30.     super(_id, 'Accounting');
  31.     this.last_report = reports[0];
  32.   }
  33.  
  34.   describe() {
  35.     console.log(`Department Accounting #(${this._id}): ${this.name}`);
  36.   }
  37.  
  38.   // other class methods...
  39. }
  40.  
  41. // called without instatiation with the 'new' keyword
  42. const year = Department.fiscal_year;
  43. console.log(year);
Singletons and Private constructors

A singleton class is one that has been restricted to a 'single' instance. This is useful when exactly one object is needed to coordinate actions throughout our logic. In TypeScript we create a singleton by adding the private modifier to the class constructor which means the class can only be instantiated within itself. To make this work we need a static method that will be used to access the single instance of the class. 

  1. class Accounting extends Department {
  2.   private last_report: string;
  3.   private static instance: Accounting; // create a private property with a type of the class
  4.  
  5.   private constructor(_id: string, public auditors: string[], private reports: string[] = ['Audit 2020']) {
  6.     super(_id, 'Accounting');
  7.     this.last_report = reports[0];
  8.   }
  9.  
  10.   /**
  11.    * Check whether the class has already been
  12.    * instantiated and return the instance.
  13.    * If not instantiated, create a new instance
  14.    */
  15.   static getInstance() {
  16.     if (this.instance) {
  17.       return this.instance;
  18.     }
  19.  
  20.     this.instance = new Accounting('dahquaitah2ohph0eevooc8k', ['Jane', 'Mary']);
  21.     return this.instance;
  22.   }
  23.  
  24.   // other class logic...
  25. }
  26.  
  27. const accounting = Accounting.getInstance();
  28. accounting.addEmployee('Jake');
  29. accounting.addEmployee('George');
  30. accounting.addReports('Taxes 2021');
Interfaces

In TypeScript, an interface is defined similarly to an object type. They define the structure of an object. 

  1. interface Person {
  2.   name: string;
  3.   age: number;
  4.  
  5.   greet(phrase: string): void;
  6. }
  7.  
  8. let user: Person;
  9.  
  10. user = {
  11.   name: 'Peter',
  12.   age: 27,
  13.   greet(phrase: string) {
  14.     console.log(`${phrase}, ${this.name} here`);
  15.   }
  16. };
  17.  
  18. user.greet('Hi there');
Interfaces with classes

Just like abstract classes, interfaces can be used to define what multiple classes should implement/overide. This makes sure that classes share the same functionality. Here we use the implements keyword instead of extends. We can also define the properties of the interface as readonly. For optional properties we use the ? symbol after the property name like so height?: number. 

  1. interface Greetable {
  2.   name: string;
  3.   greet(phrase: string): void;
  4. }
  5.  
  6. class Person implements Greetable {
  7.   constructor(public name: string, age: number) {
  8.  
  9.   }
  10.  
  11.   greet(phrase: string) {
  12.     console.log(`${phrase}, ${this.name} here`);
  13.   }
  14. }
  15.  
  16. /**
  17.  * We can use an interface as a type for a variable
  18.  */
  19. let user: Greetable;
  20.  
  21. /**
  22.  * This is correct because the Person() class is 
  23.  * based on the Greetable interface
  24.  */
  25. user = new Person('Peter', 27);
  26. user.greet('Hi there');
Inheritance in Interfaces

Just like classes interfaces can inherit properties from one another. This is done through the extends keyword and the difference is that interfaces can extend multiple interfaces defined by comma separated declaration. Note: This is not possible for classes. 

  1. interface Named {
  2.   readonly name: string;
  3. }
  4.  
  5. interface Greetable extends Named {
  6.   height ? : number;
  7.   greet(phrase: string): void;
  8. }
Interfaces as function types

We can as well use interfaces as a function type. Here we define it as a set of parameters with a return type. 

  1. interface AddFunction {
  2.   (number_one: number, number_two: number): number;
  3. }
  4.  
  5. let add: AddFunction = (a: number, b: number) => {
  6.   return a + b;
  7. };
  8.  
  9. console.log(`Add Result: ${add(5, 6)}`);
