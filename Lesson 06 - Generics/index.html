Generics

Generics

In this section I learned about Generics and how to define them. Generics are reusable components that give us the ability to create our own types that can be reused in a variety of ways. Read more here →

Built in Generics

There are a couple of built in generics such as Array and Promise. These generics allow us to give TypeScript control over checking the types returned in variables using these generics. For example we can define the Array generic where we can pass what kind of values are contained in an array. 

  1. // same as defining the type as: string[]
  2. const names: Array<string> = ['two', 'three', '4.5', '6'];

Promises are another built in generic where we can define the type of whatever is passsed to the resolve, reject callbacks. 

  1. const promise: Promise<string> = new Promise((resolve, reject) => {
  2.   setTimeout(() => {
  3.     resolve('This is done');
  4.   }, 2000);
  5. });
Generic Functions

We can create a generic function where we would be flexible with the types of the arguments passed and their values. In the below example, TypeScript would infer the return type as an intersection type of the defined generic types. 

  1. /**
  2.  * Gives us the flexibility to allow a variety of types
  3.  * passed and defined diffrently when we call the function
  4.  */
  5. const merge = <T, U>(first_obj: T, second_obj: U) => {
  6.   return Object.assign(first_obj, second_obj);
  7. };
  8.  
  9. const merged_one = merge({
  10.   name: 'Peter'
  11. }, {
  12.   age: 30
  13. });
  14. console.log(merged_one.name);
  15.  
  16. const merged_two = merge({
  17.   name: 'Peter',
  18.   hobbies: ['cycling', 'swimming']
  19. }, {
  20.   age: 30,
  21.   height: '6 feet'
  22. });
  23. console.log(merged_two.hobbies);
Generic Constraints

Generic constraints are used to tell TypeScript what to expect as properties or types for our generics. We use the extends keyword and pass the type that constrains our generic to the type definition. This can be an interface, union type e.t.c. and even common JS types. 

  1. /**
  2.  * Gives us the flexibility to allow a variety of types
  3.  * passed and defined diffrently when we call the function
  4.  * 
  5.  * Constraints allow us to lock down what kind of type our generic type will have.
  6.  * Below we constrain our generic function to object arguments.
  7.  */
  8. const merge = <T extends object, U extends object>(first_obj: T, second_obj: U) => {
  9.   return Object.assign(first_obj, second_obj);
  10. };
  11.  
  12. const merged_one = merge({
  13.   name: 'Peter'
  14. }, {
  15.   age: 30
  16. });
  17. console.log(merged_one.name);
  18.  
  19. const merged_two = merge({
  20.   name: 'Peter',
  21.   hobbies: ['cycling', 'swimming']
  22. }, {
  23.   age: 30,
  24.   height: '6 feet'
  25. });
  26. console.log(merged_two.hobbies);
More on Generic Functions

Another example would be where we dont care about the types passed so long as they have a length property. We can do this by adding an interface as a constraint with length as a property definition. 

  1. interface Lengthy {
  2.   length: number;
  3. }
  4.  
  5. // here we can also define the return type as a 'tuple' containing the generic type and a string
  6. const countAndDescribe = <Type extends Lengthy> (element: Type): [Type, string] => {
  7.   let description = 'Got no value.';
  8.  
  9.   if (element.length > 0) {
  10.     if (element.length === 1) {
  11.       description = 'Got 1 value.';
  12.     } else {
  13.       description = `Got ${element.length} values.`;
  14.     }
  15.   }
  16.  
  17.   return [element, description];
  18. };
  19.  
  20. console.log(countAndDescribe('Hi there!'));
  21. console.log(countAndDescribe(['Hi there!', 'Welcome home!']));
The 'keyof' constraint

The keyof constraint is useful whenever we want to tell TypeScript that a parameter in our generic function relies on another. In the example below we know that the second parameter is going to be a field in the first parameter and we add the keyof constraint so that TypeScript can check for this property. 

  1. const extractAndConvert = <T extends object, U extends keyof T>(obj: T, key: U) => {
  2.   return `Value: ${obj[key]}`;
  3. };
  4.  
  5. extractAndConvert({
  6.   name: 'Peter'
  7. }, 'name');
Generic classes

Generic classes are very useful especially when we want flexibility of how data is handled and manipulated in our classes. In the example below we want the data handled to be any primitive type where we store these values to an array. We can also define generic functions in the class with their own generic types. More on this here → 

  1. class ExampleStorage<Type extends string | number | boolean> {
  2.   private data: Type[] = [];
  3.  
  4.   addItem(item: Type) {
  5.     this.data.push(item);
  6.   }
  7.  
  8.   removeItem(item: Type) {
  9.     this.data.splice(this.data.indexOf(item), 1);
  10.   }
  11.  
  12.   getItems() {
  13.     return [...this.data];
  14.   }
  15. }
  16.  
  17. const sweets = new ExampleStorage<string>();
  18. sweets.addItem('Cake');
  19. sweets.addItem('Cookies');
  20. console.log(`Sweets: ${JSON.stringify(sweets.getItems())}`);
  21.  
  22. const scores = new ExampleStorage<number>();
  23. scores.addItem(10);
  24. scores.addItem(46);
  25. scores.addItem(90);
  26. console.log(`Math Test Scores: ${JSON.stringify(scores.getItems())}`);
Generic Utility Types

In TypeScript, there are a couple of utility types that set certain constraints while giving us the flexibility to define how data is set and used. These types are well document here →. Below are examples of the Partial and Readonly utility types. 

  1. interface CourseGoal {
  2.   title: string;
  3.   description: string;
  4.   complete_until: Date;
  5. }
  6.  
  7. const createCourseGoal = (title: string, description: string, date: Date): CourseGoal => {
  8.   let goal: Partial<CourseGoal> = {};
  9.   goal.title = title;
  10.   goal.description = description;
  11.   goal.complete_until = date;
  12.  
  13.   return goal as CourseGoal;
  14. };
  15.  
  16. // This means we cannot do something like: usernames.push('James');
  17. const usernames: Readonly<string[]> = ['Peter', 'John'];
