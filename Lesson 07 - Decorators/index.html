Defining a Decorator

Decorators are defined as functions which are then added before a class definition, method, property, accessor and even parameters. A decorator defined before a class is usually applied to its constructor. It is defined using the syntax @expression where expression is the name of the function. 

  1. const Logger = (constructor: Function) => {
  2.   console.log('Logging...');
  3.   console.log(constructor);
  4. };
  5.  
  6. @Logger
  7. class Person {
  8.   public name: string = 'Peter';
  9.  
  10.   constructor() {
  11.     console.log('Creating person object...');
  12.   }
  13. }
  14.  
  15. const person = new Person();
  16.  
  17. console.log(person);
Decorator Factories

Decorator factories are functions that will return an the decorator expression when called at runtime. They give us the flebility of defining logic that will be executed when the decorator is called. This way we are also able to pass arguments that will be used during the setup of the decorator. 

  1. const Logger = (log_string: string) => {
  2.   return (constructor: Function) => {
  3.     console.log(log_string);
  4.     console.log(constructor);
  5.   };
  6. };
  7.  
  8. @Logger('Logging Person...')
  9. class Person {
  10.   public name: string = 'Peter';
  11.  
  12.   constructor() {
  13.     console.log('Creating person object...');
  14.   }
  15. }
  16.  
  17. const person = new Person();
  18.  
  19. console.log(person);
Example use case of a Decorator

An example where a decorator would be useful is when we want to render data from a class onto a webpage. In the example below we have a div with an id of app where we will render the name of the person object to a h1 element added to the DOM by our decorator. This approach is used a lot in Angular to render components. 

  1. const withTemplate = (template: string, selector: string) => {
  2.   return (constructor: any) => {
  3.     const el = document.getElementById(selector);
  4.     const context_person = new constructor();
  5.  
  6.     if (el) {
  7.       el.innerHTML = template;
  8.       el.querySelector('h1')!.textContent = `Person object name: ${context_person.name}`;
  9.     }
  10.   }
  11. };
  12.  
  13. @withTemplate('<h1></h1>', 'app')
  14. class Person {
  15.   public name: string = 'Peter';
  16.  
  17.   constructor() {
  18.     console.log('Creating person object...');
  19.   }
  20. }
  21.  
  22. const person = new Person();
Property Decorators

A property decorator is declared before a property declaration. It receives two arguments which are:

  1. The constructor of the class when declared before a static property of the class or the prototype of the class when the property is an instance member.
  2. The property name

  1. const Log = (target: any, property_name: string) => {
  2.   console.log('Property Decorator');
  3.   console.log(target, property_name);
  4. };
  5.  
  6. class Product {
  7.   @Log
  8.   public title: string;
  9.  
  10.   // other class logic...
  11. }
Parameter Decorators

A parameter decorator is declared before a parameter declaration in a function. It receives three arguments which are:

  1. The constructor of the class when declared before a static parameter of a method in the class or the prototype of the class when the parameter is an instance member.
  2. The method name
  3. The index of the parameter in the method's parameter list

  1. const Log = (target: any, method_name: string, position: number) => {
  2.   console.log('Parameter Decorator');
  3.   console.log(target);
  4.   console.log(method_name);
  5.   console.log(position);
  6. };
  7.  
  8. class Product {
  9.   private _price: number;
  10.  
  11.   constructor(
  12.     _price: number
  13.   ) {
  14.     this._price = _price;
  15.   }
  16.  
  17.   getPriceWithTax(@Log tax: number) {
  18.     return this._price * (1 + tax);
  19.   }
  20. }
Accessor Decorators

An accessor decorator is declared before an accessor declaration such as getters and setters. It receives three arguments which are:

  1. The constructor of the class when declared before a static accessor of the class or the prototype of the class when the accessor is an instance member.
  2. The accessor name
  3. The property descriptor of the accessor

  1. const Log = (target: any, accessor_name: string, descriptor: PropertyDescriptor) => {
  2.   console.log('Accessor Decorator');
  3.   console.log(target);
  4.   console.log(accessor_name);
  5.   console.log(descriptor);
  6. };
  7.  
  8. class Product {
  9.   private _price: number;
  10.  
  11.   @Log
  12.   set price(val: number) {
  13.     if (val > 0) {
  14.       this._price = val;
  15.     } else {
  16.       throw new Error('Invalid price - should be positive');
  17.     }
  18.   }
  19.  
  20.   // other class logic...
  21. }
Method Decorators

A method decorator is declared before an method declaration. It receives three arguments which are:

  1. The constructor of the class when declared before a static method of the class or the prototype of the class when the method is an instance member.
  2. The method name
  3. The property descriptor of the method

  1. const Log = (target: any, method_name: string, descriptor: PropertyDescriptor) => {
  2.   console.log('Method Decorator');
  3.   console.log(target);
  4.   console.log(method_name);
  5.   console.log(descriptor);
  6. };
  7.  
  8. class Product {
  9.   private _price: number;
  10.  
  11.   constructor(
  12.     _price: number
  13.   ) {
  14.     this._price = _price;
  15.   }
  16.  
  17.   @Log
  18.   getPriceWithTax(tax: number) {
  19.     return this._price * (1 + tax);
  20.   }
  21. }
Returning a Class in a class decorator

We can return a class in our class decorator which would give us the flexibility to pass more logic to the constructor of the class. This inturn replaces the constructor which means we would need to pass the arguments as expected in the constructor. We also extend the constructor and call super() so as to instantiate the logic of the original constructor. This decorator will now only execute when the class is instantiated. 

  1. const withTemplate = (template: string, selector: string) => {
  2.   /**
  3.    * Lets TypeScript know that the factory returns an implementation of a class
  4.    * with a name property
  5.    */
  6.   return <Type extends { new(...args: any[]): {name: string} }>(original_constructor: Type) => {
  7.     /**
  8.      * Extends the class the decorator is applied to
  9.      */
  10.     return class extends original_constructor {
  11.       /**
  12.        * Replaces the constructor in the class but still instantiates
  13.        * all the logic from that class constructor
  14.        */
  15.       constructor(...args: any[]) {
  16.         super();
  17.  
  18.         console.log('Render Person');
  19.         const el = document.getElementById(selector);
  20.  
  21.         if (el) {
  22.           el.innerHTML = template;
  23.           el.querySelector('h1') !.textContent = `Person object name: ${this.name}`;
  24.         }
  25.       }
  26.     };
  27.   }
  28. };
  29.  
  30. @withTemplate('<h1></h1>', 'app') // means that the decorator will only be executed when the class is instantiated
  31. class Person {
  32.   public name: string = 'Peter';
  33.  
  34.   constructor() {
  35.     console.log('Creating person object...');
  36.   }
  37. }
  38.  
  39. const person = new Person();
  40.  
  41. console.log(person);
Using a method decorator to modify how a method is accessed and executed

Here we modify the method's property descriptor by adding an extra layer(get()) which will bind the context of this to the method the decorator is attached to. 

  1. const Autobind = (target: any, method_name: string, descriptor: PropertyDescriptor) => {
  2.   const original_method = descriptor.value;
  3.   const adjusted_descriptor: PropertyDescriptor = {
  4.     configurable: true,
  5.     enumerable: false,
  6.     get() {
  7.       /**
  8.        * Makes sure that 'this' is always bound to the method the decorator
  9.        * is attached to
  10.        */
  11.       const bound_function = original_method.bind(this);
  12.       return bound_function;
  13.     }
  14.   };
  15.  
  16.   return adjusted_descriptor;
  17. };
  18.  
  19. class Printer {
  20.   public message: string = 'This works!';
  21.  
  22.   @Autobind
  23.   showMessage() {
  24.     console.log(this.message);
  25.   }
  26. }
  27.  
  28. const printer = new Printer();
  29.  
  30. const button = <HTMLButtonElement>document.querySelector('button')!;
  31. button.addEventListener('click', printer.showMessage);
