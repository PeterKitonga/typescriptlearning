Node.js + Express & TypeScript

In this section I learned how to use TypeScript with Node.js and Express. This offers type safety during development of node.js applications. Node does not support TypeScript out of the box but with a little of configuration we can start working TypeScript in Node environments.

Express Setup

To ensure we work with TypeScript, we need to install type definitions for express and node(npm install --save-dev @types/express @types/node). These ensure we add types for express which will allow us to create express middlewares with TypeScript. With this in place we can now modify the app.ts file to look like below. 

  1. require('dotenv-expand')(require('dotenv').config());
  2.  
  3. import path from 'path';
  4. import express, {
  5.   NextFunction,
  6.   Request,
  7.   Response
  8. } from 'express';
  9.  
  10. const app = express();
  11. const port = process.env.APP_PORT;
  12.  
  13. app.use(express.json());
  14. app.use(express.static(path.join(__dirname, '../public')));
  15.  
  16. app.use((req: Request, res: Response, next: NextFunction) => {
  17.   res.setHeader('Access-Control-Allow-Origin', '*');
  18.   res.setHeader('Access-Control-Allow-Headers', 'Content-Type, Authorization');
  19.   res.setHeader('Access-Control-Allow-Methods', 'GET, POST, PUT, PATCH, DELETE');
  20.  
  21.   next();
  22. });
  23.  
  24. app.use((err: Error, req: Request, res: Response, next: NextFunction) => {
  25.   res.status(500).json({
  26.     message: err.message
  27.   });
  28. });
  29.  
  30. app.get('/', (req: Request, res: Response, next: NextFunction) => {
  31.   res.status(200).json({
  32.     message: 'Hello!'
  33.   });
  34. });
  35.  
  36. // mount the nodejs app to a port
  37. app.listen(port, () => {
  38.   console.log(`Server running at: ${process.env.APP_BASE_URL}`);
  39.   console.log('Hit CTRL-C to stop the server');
  40. });
Development server

The webpack-dev-server doesn't work quite well with express and to deal with that we can use nodemon to serve the application in development mode. Here we can use the nodemon-webpack-plugin(npm install nodemon-webpack-plugin --save-dev) dependency which watches for changes in the output file configured on webpack. This however only works with webpack cli in --watch mode in our npm scripts. 

  1. {
  2.   "scripts": {
  3.     "serve": "webpack --watch --node-env development",
  4.     "build": "webpack --node-env production"
  5.   }
  6. }

We can now configure the nodemon webpack plugin in the webpack.config.js file as a plugin. Like so: 

  1. // webpack.config.js
  2. const NodemonPlugin = require('nodemon-webpack-plugin');
  3. const {
  4.   CleanWebpackPlugin
  5. } = require('clean-webpack-plugin');
  6.  
  7. module.exports = {
  8.   // other configs...
  9.   /**
  10.    * Let's webpack know which environment to target
  11.    * 
  12.    * @link https://webpack.js.org/configuration/target/#target
  13.    */
  14.   target: 'node',
  15.   plugins: [
  16.     /**
  17.      * This plugin cleans up the output path before doing a re-build in production.
  18.      * 
  19.      * @link https://www.npmjs.com/package/clean-webpack-plugin
  20.      */
  21.     new CleanWebpackPlugin(),
  22.     /**
  23.      * This plugin uses Nodemon to watch and restart the module's output file.
  24.      * 
  25.      * @link https://www.npmjs.com/package/nodemon-webpack-plugin
  26.      */
  27.     new NodemonPlugin()
  28.   ]
  29. };
Try it

To run the app, in your terminal run the following commands: 

  1. cd Lesson\ 12\ -\ Node\|Express\|TypeScript/app/
  2.  
  3. npm install // if you have not installed dependencies
  4.  
  5. npm run serve // starts application
  6.  
  7. npm run build // builds application for production
