Modules & Namespaces

Modules & Namespaces

In this section I learned how to use modules and namespacing. This helps organize code into seperate files that are then bundled together during compilation.

Using Namespaces

Namespaces help organize our code through bundling code in similar groups. We use the namespace keyword to define a namespace and pass the name of the after the keyword. To allow TypeScript to find the types defined that depend on each other, we can place them in the same namespace. Below is an example where we have interfaces in a sperate file and well as the Project in its own file. We need to export these types so as to be able to access them in the different files. We also use the three slashes at the top of the file to import files of the same namespace. 

  1. // src/drag-drop-interfaces.ts
  2. namespace App {
  3.   // Drag & Drop interfaces
  4.   export interface Draggable {
  5.     dragStartHandler(event: DragEvent): void;
  6.     dragEndHandler(event: DragEvent): void;
  7.   }
  8.  
  9.   export interface DragTarget {
  10.     dragOverHandler(event: DragEvent): void;
  11.     dropHandler(event: DragEvent): void;
  12.     dragLeaveHandler(event: DragEvent): void;
  13.   }
  14. }
  15.  
  16. // src/project-model.ts
  17. namespace App {
  18.   // Project statuses
  19.   export enum ProjectStatus {
  20.     ACTIVE = 'active',
  21.       FINISHED = 'finished',
  22.   }
  23.  
  24.   /**
  25.    * Project type
  26.    * Used as the type definition for all projects
  27.    */
  28.   export class Project {
  29.     constructor(
  30.       public id: string,
  31.       public title: string,
  32.       public description: string,
  33.       public people: number,
  34.       public status: ProjectStatus
  35.     ) {}
  36.   }
  37. }
  38.  
  39. // src/app.ts
  40. /// <reference path="drag-drop-interfaces.ts" />
  41. /// <reference path="project-model.ts" />
  42.  
  43. namespace App {
  44.   // app logic...
  45. }

Since JavaScript, has no way of knowing how to import the different files, we need to modify the tsconfig.json file where we change the "outFile": "./" config to "outFile": "./dist/bundle.js" this way we can bundle all the code into a single file. This also means we have to change the "module" setting to "module": "amd" since this module code generation is only available for amd.

Using Modules

To use modules we first need to modify our tsconfig.json file and specify our module as "module": "es2015". This means that we no longer need the "outFile": "./dist/bundle.js" configuration and should be commented out. With this in place, we can use modules where we export all variables, types, classes that are to be used in other files. This would allow us to use the keyword import at the top of any file to import code that is needed in the respective file. For example, below we are importing the Project class and ProjectStatus enum type to the state management file. 

  1. // src/models/project.ts
  2. // Project statuses
  3. export enum ProjectStatus {
  4.   ACTIVE = 'active',
  5.     FINISHED = 'finished',
  6. }
  7.  
  8. /**
  9.  * Project type
  10.  * Used as the type definition for all projects
  11.  */
  12. export class Project {
  13.   constructor(
  14.     public id: string,
  15.     public title: string,
  16.     public description: string,
  17.     public people: number,
  18.     public status: ProjectStatus
  19.   ) {}
  20. }
  21.  
  22. // src/state/project.ts
  23. import {
  24.   Project,
  25.   ProjectStatus
  26. } from '../models/project.js';
  27.  
  28. type Listener < Type > = (items: Type[]) => void;
  29.  
  30. class State < Type > {
  31.   protected listeners: Listener < Type > [] = [];
  32.  
  33.   /**
  34.    * Responsible for adding all listeners that will run when a project is added
  35.    */
  36.   addListener(listener_function: Listener < Type > ) {
  37.     this.listeners.push(listener_function);
  38.   }
  39. }
  40.  
  41. /**
  42.  * Singleton class
  43.  * Handles state management of added projects
  44.  */
  45. export class ProjectState extends State < Project > {
  46.   private projects: Project[] = [];
  47.   private static instance: ProjectState;
  48.  
  49.   private constructor() {
  50.     super();
  51.   }
  52.  
  53.   /**
  54.    * Makes sure that only one instance of the class is created
  55.    */
  56.   static getInstance() {
  57.     if (!this.instance) {
  58.       this.instance = new ProjectState();
  59.     }
  60.  
  61.     return this.instance;
  62.   }
  63.  
  64.   // other class logic...
  65. }
  66.  
  67. /**
  68.  * Instantiates the state management object for use in all logic
  69.  */
  70. export const project_state = ProjectState.getInstance();
Try it

To run the app, in your terminal run the following commands: 

  1. cd Lesson\ 08\ -\ Modules\|Namespaces/app/
  2.  
  3. npm install // if you have not installed dependencies
  4.  
  5. npm run compile:type // compiles typescript to common js
  6.  
  7. npm run serve // starts application
