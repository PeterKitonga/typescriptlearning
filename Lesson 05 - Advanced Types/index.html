Advanced Types

Advanced Types

In this section I learned about advance types available in TypeScript such as indexed properties, type casting, function overloads and more.

Intersection Types

Intersection types allow us to combine multiple types into a single type. They work much like extending interfaces but can be used to combine other types together as well. 

  1. type Admin = {
  2.   name: string;
  3.   permissions: string[];
  4. };
  5.  
  6. type Employee = {
  7.   name: string;
  8.   start_date: Date;
  9. };
  10.  
  11. // works the same as interfaces extending each other
  12. type ElevatedEmployee = Admin & Employee;
  13.  
  14. const employee_one: ElevatedEmployee = {
  15.   name: 'Peter',
  16.   permissions: ['create-server'],
  17.   start_date: new Date()
  18. };
  19.  
  20. type Combinable = string | number;
  21. type Numeric = number | boolean;
  22.  
  23. // can also be used to combine union types
  24. type Universal = Combinable & Numeric;
Type Guards

Type guards assist in checking the type of value passed to a variable. They are especially useful when checking variables with union types. In the example below we use typeof to check the type of value passed to the add() function. 

  1. type Combinable = string | number;
  2.  
  3. const add = (a: Combinable, b: Combinable) => {
  4.   // here we use the 'typeof' type guard
  5.   if (typeof a === 'string' || typeof b === 'string') {
  6.     return a.toString() + b.toString();
  7.   } else {
  8.     return a + b;
  9.   }
  10. }

We can also use in as a type guard to check whether a property exists in an object. Consider the example below where we have a union type of Employee or Admin. Since some properties are not common between them, we need to check the value passed in our function for possible properties. 

  1. type Admin = {
  2.   name: string;
  3.   permissions: string[];
  4. };
  5.  
  6. type Employee = {
  7.   name: string;
  8.   start_date: Date;
  9. };
  10.  
  11. type UnknownEmployee = Employee | Admin;
  12.  
  13. const printEmployee = (emp: UnknownEmployee) => {
  14.   console.log(`Name: ${emp.name}`);
  15.  
  16.   // here we use the 'in' type guard
  17.   if ('permissions' in emp) {
  18.     console.log(`Permissions: ${JSON.stringify(emp.permissions)}`);
  19.   }
  20.  
  21.   if ('start_date' in emp) {
  22.     console.log(`Start Date: ${emp.start_date.toLocaleDateString()}`);
  23.   }
  24. };
  25.  
  26. printEmployee({
  27.   name: 'George',
  28.   start_date: new Date()
  29. });

Another type guard is instanceof which can be used to check if the value passed is an instance of a certain class. 

  1. class Car {
  2.   drive() {
  3.     console.log('Driving...');
  4.   }
  5. }
  6.  
  7. class Truck {
  8.   drive() {
  9.     console.log('Driving a truck...');
  10.   }
  11.  
  12.   loadCargo(amount: number) {
  13.     console.log(`Loading '${amount}' cargo...`);
  14.   }
  15. }
  16.  
  17. type Vehicle = Car | Truck;
  18.  
  19. const vehicle_one = new Car();
  20. const vehicle_two = new Truck();
  21.  
  22. const useVehicle = (vehicle: Vehicle) => {
  23.   vehicle.drive();
  24.  
  25.   // here we use the 'instanceof' type guard
  26.   if (vehicle instanceof Truck) {
  27.     vehicle.loadCargo(1000);
  28.   }
  29. };
  30.  
  31. useVehicle(vehicle_two);
Discriminated Unions

Discriminated unions are used together with object(as well as interfaces) and union types. This involves passing a common property that is a literal type. This property is then check for the literal type value passed and apply the respective logic. 

  1. interface Bird {
  2.   kind: 'bird';
  3.   flying_speed: number;
  4. }
  5.  
  6. interface Horse {
  7.   kind: 'horse';
  8.   running_speed: number;
  9. }
  10.  
  11. type Animal = Bird | Horse;
  12.  
  13. const moveAnimal = (animal: Animal) => {
  14.   let speed: number;
  15.  
  16.   // checks the literal type value passed
  17.   switch (animal.kind) {
  18.     case 'bird':
  19.       speed = animal.flying_speed;
  20.       break;
  21.     case 'horse':
  22.       speed = animal.running_speed;
  23.       break;
  24.   }
  25.  
  26.   console.log(`Moving at speed ${speed} km/h`);
  27. };
  28.  
  29. moveAnimal({
  30.   kind: 'bird',
  31.   flying_speed: 20.99
  32. });
Type Casting

Type casting helps us tell TypeScript that some value is of a specific type where it is not able to detect on its own. This gives us developers the responsibility to let TypeScript know what type we are dealing with. For example, below TypeScript would only know that a html element with the specified selector exists but not what kind of html element it is. 

  1. /**
  2.  * Below can also be written as:
  3.  * const userInputElement = document.getElementById('user-input')! as HTMLInputElement;
  4.  * 
  5.  * The '!' signifies that the type will never yield null
  6.  */
  7. const userInputElement = < HTMLInputElement > document.getElementById('user-input') !;
  8.  
  9. userInputElement.value = 'Hi there!';
Indexed properties

These are useful when we don't know all the names of a type's(mostly object types) properties. They control how we name properties and values of the properties. In the example below we restrict all property names to strings and they should yield values that are strings as well. Read more here → 

  1. interface ErrorContainer {
  2.   [prop: string]: string;
  3. }
  4.  
  5. const error_bag: ErrorContainer = {
  6.   email: 'Not a valid email!',
  7.   username: 'Must be more than 3 characters'
  8. };
Function Overloads

Function overloads allow us to define different signatures of a single function. This can be used to let TypeScript know what type will be returned depending on the parameter types passed. 

  1. type Combinable = string | number;
  2. type Numeric = number | boolean;
  3.  
  4. type Universal = Combinable & Numeric;
  5.  
  6. function add(a: number, b: number): number;
  7.  
  8. function add(a: string, b: number): string;
  9.  
  10. function add(a: number, b: string): string;
  11.  
  12. function add(a: string, b: string): string;
  13.  
  14. function add(a: Combinable, b: Combinable) {
  15.   if (typeof a === 'string' || typeof b === 'string') {
  16.     return a.toString() + ' ' + b.toString();
  17.   } else {
  18.     return a + b;
  19.   }
  20. }
  21.  
  22. const result = add('Peter', 'Kitonga');
  23. console.log(result);
Optional Chaining

Just like how we define optional properties with the ? symbol, we can also use the symbol to check whether a property exists in the object being accessed. This helps prevent undefined errors where we might be trying to access a property that might not exist. 

  1. const fetched_user = {
  2.   id: 'eiveng8phak6hoh4aeth4iesheig7cho',
  3.   name: 'Peter',
  4.   // job: { title: 'Software Engineer', description: 'Making software for humans' }
  5. };
  6.  
  7. console.log(fetched_user?.job?.title);
Nullish Coalescing

Nullish coalescing is useful whenever we want to handle how a variable that is null or undefined is treated. Here we use double question marks(the ?? symbol) before the default value and will treat an empty string as a value. 

  1. const user_input = '';
  2. const stored_data = user_input ?? 'DEFAULT';
  3.  
  4. console.log(stored_data);
