3rd Party Libraries & TypeScript

In this section I learned how to use 3rd party Libraries in my TypeScript projects. Not all libraries have support for TypeScript out of the box and require extra configuration for them to have TypeScript support.

Using JavaScript libraries with TypeScript

Some popular libraries do not have out of the box support for TypeScript. This present an issue whenever we want to use these libraries in our TypeScript projects. A workaround for this is to search for @types dependencies for these libraries which add support for TypeScript. For example when installing lodash, we would first install the JavaScript library the usual way with npm(npm install --save lodash) and then install @types/lodash as a dev dependency(npm install --save-dev @types/lodash) 

  1. import _ from 'lodash';
  2.  
  3. let shuffled = _.shuffle([1, 2, 3, 4]);
  4. console.log(shuffled);
Using "declare"

There are moments where we have JavaScript libraries or files with global variables and we want access to these variables. We can use the declare keyword to let TypeScript know how to handle these variables so that we can use them like so: 

  1. <!-- index.html -->
  2. <script>
  3.   let GLOBAL = 'Something global';
  4. </script>
  5.  
  1. declare let GLOBAL: any;
  2.  
  3. console.log(GLOBAL);
Class transformation

Sometimes we want to convert plain JavaScript objects into instances of a class with TypeScript support. This can be done using a 3rd library called class-transformer(npm install --save class-transformer). This library gives us functionality that allows us to transform these objects to instances of a class or vice versa. We first begin by creating a class we would like to convert to like the below class for Product. Note: A required dependency for this library is the reflect-metadata(npm install --save reflect-metadata) library. 

  1. // product.model.ts
  2. export class Product {
  3.   title: string;
  4.   price: number;
  5.  
  6.   constructor(title: string, price: number) {
  7.     this.title = title;
  8.     this.price = price;
  9.   }
  10.  
  11.   getInformation() {
  12.     return [this.title, `$${this.price}`];
  13.   }
  14. }

With that in place we can then import the class-transformer library and use the plainToClass method to convert an array of objects of our products. This will give us access to all functionality of the Product class instance where we can call the getInformation method. 

  1. // app.ts
  2. import 'reflect-metadata';
  3. import { Product } from './product.model';
  4. import {
  5.   plainToClass
  6. } from 'class-transformer';
  7.  
  8. const products = [{
  9.     title: 'A carpet',
  10.     price: 29.99
  11.   },
  12.   {
  13.     title: 'A book',
  14.     price: 10.99
  15.   }
  16. ];
  17.  
  18. const loaded_products = plainToClass(Product, products);
  19.  
  20. for (const prod of loaded_products) {
  21.   console.log(prod.getInformation());
  22. }
Class validation

Another useful 3rd party library is class-validator(npm install class-validator --save). This library offers a tone of functionality for validation of class properties with decorators. It internally uses the famous validator.js library. 

  1. // product.model.ts
  2. import {
  3.   IsNotEmpty,
  4.   IsNumber,
  5.   IsPositive
  6. } from 'class-validator';
  7.  
  8. export class Product {
  9.   @IsNotEmpty({
  10.     message: 'Title field cannot be empty.'
  11.   })
  12.   title: string;
  13.  
  14.   @IsNumber({
  15.     maxDecimalPlaces: 2
  16.   }, {
  17.     message: 'Price field should be a number.'
  18.   })
  19.   @IsPositive({
  20.     message: 'Price field should be a positive number.'
  21.   })
  22.   price: number;
  23.  
  24.   constructor(title: string, price: number) {
  25.     this.title = title;
  26.     this.price = price;
  27.   }
  28.  
  29.   getInformation() {
  30.     return [this.title, `$${this.price}`];
  31.   }
  32. }
  33.  
  34. // app.ts
  35. import {
  36.   Product
  37. } from './product.model';
  38. import {
  39.   validate
  40. } from 'class-validator';
  41.  
  42. const new_prod = new Product('', -12.99);
  43.  
  44. validate(new_prod).then(errors => {
  45.   if (errors.length > 0) {
  46.     console.log('Validation errors present');
  47.     console.log(errors);
  48.   } else {
  49.     console.log(new_prod.getInformation());
  50.   }
  51. });
Try it

To run the app, in your terminal run the following commands: 

  1. cd Lesson\ 10\ -\ Libraries\|TypeScript/app/
  2.  
  3. npm install // if you have not installed dependencies
  4.  
  5. npm run compile:type // compiles typescript to common js
  6.  
  7. npm run serve // starts application
